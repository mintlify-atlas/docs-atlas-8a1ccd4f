The DEF-DEV Terminal is an interactive command-line interface for controlling Urbanshade OS, managing localStorage, triggering system states, and running package manager commands.

<Note>
  Access the terminal from **DEF-DEV → Core → Terminal**, or open DEF-DEV with `Ctrl+Shift+D`.
</Note>

## Core commands

| Command | Description | Example |
|---|---|---|
| `help` | Show all available commands | `help` |
| `help [cmd]` | Show help for a specific command | `help crash` |
| `clear` | Clear terminal output | `clear` |
| `echo [msg]` | Print a message | `echo Hello World` |
| `date` | Show current date/time | `date` |
| `uptime` | Show system uptime | `uptime` |
| `whoami` | Show current user info | `whoami` |
| `status` | Show system status | `status` |

## System commands

| Command | Description | Example |
|---|---|---|
| `crash [type]` | Queue a crash screen | `crash KERNEL_PANIC` |
| `bugcheck [code] [desc]` | Queue a bugcheck BSOD | `bugcheck FATAL_ERROR Test` |
| `reboot` | Queue system reboot | `reboot` |
| `shutdown` | Queue system shutdown | `shutdown` |
| `lockdown [protocol]` | Trigger lockdown mode | `lockdown ALPHA` |
| `recovery` | Boot into recovery mode | `recovery` |
| `wipe --confirm` | Wipe all data (dangerous) | `wipe --confirm` |

```bash
# Queue a bugcheck with a custom code
$ bugcheck DATA_INCONSISTENCY Database validation failed
Bugcheck queued: DATA_INCONSISTENCY

# Trigger lockdown protocol
$ lockdown GAMMA
Lockdown protocol GAMMA queued
```

## Storage commands

| Command | Description | Example |
|---|---|---|
| `ls [filter]` | List localStorage keys | `ls settings` |
| `get <key>` | Get a storage value | `get urbanshade_theme` |
| `set <key> <value>` | Queue setting a value | `set debug true` |
| `del <key>` | Queue deleting a key | `del temp_data` |

```bash
# List keys containing 'settings'
$ ls settings
settings_theme
settings_volume
settings_notifications

# Get a specific value
$ get settings_theme
"urbanshade-dark"

# Set a value (queued for main OS)
$ set mykey myvalue
Storage write queued: mykey
```

## Utility commands

| Command | Description |
|---|---|
| `toast <type> <msg>` | Queue a toast notification (`success` / `error` / `info` / `warning`) |
| `queue` | Show pending commands in the queue |
| `exec <cmd>` | Execute a raw command |

## UUR package manager

| Command | Description |
|---|---|
| `uur` | Show UUR help |
| `uur lst` | List installed packages |
| `uur search <query>` | Search available packages |

```bash
# Search for packages
$ uur search terminal
  terminal-plus - Enhanced terminal features (v1.0.0)
  terminal-themes - Custom terminal color schemes (v1.2.0)

# List installed packages
$ uur lst
  base-system (1.0.0)
  def-dev-tools (1.0.0)
```

## Command queue system

Commands like `crash`, `reboot`, `set`, and `del` do not execute immediately. Instead, they are written to a localStorage queue:

1. You run a command in the DEF-DEV terminal.
2. The command is serialized and stored under `urbanshade_command_queue`.
3. The main OS page polls localStorage **4 times per second** for pending commands.
4. The main OS dequeues and executes each command in order.

This design allows DEF-DEV to control the main OS even when running in a separate tab or window. Use `queue` to inspect pending commands and `status` to check persistence state.

## Keyboard shortcuts

| Shortcut | Action |
|---|---|
| `Enter` | Execute command |
| `↑` / `↓` | Navigate command history |
| `Tab` | Auto-complete (where supported) |

## Dangerous commands

<Warning>
  `wipe --confirm` permanently deletes **all** localStorage data, including settings, accounts, files, and OS state. This cannot be undone. The system will require a fresh installation afterwards.
</Warning>
